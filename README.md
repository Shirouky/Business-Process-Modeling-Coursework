# Business Process Modeling Coursework

# Моделирование бизнес-процессов: Подготовка ежемесячного отчета по продажам

## Цель проекта
Оптимизация и автоматизация процесса формирования ежемесячного отчет по продажам для повышения скорости подготовки данных и снижения количества ручных операций.

## Задачи
- Провести анализ бизнес-процесса в нотации **AS-IS**
- Разработать оптимизированную модель **TO-BE** 
- Спроектировать архитектуру системы автоматизации отчетности

## Описание процесса
*Существующая ситуация (AS-IS):*
Менеджеры отделов собирают данные о продажах. Финансовый отдел проверяет корректность данных. Маркетолог анализирует результаты и формирует графики. Отчёт предоставляется руководству для принятия решений. При необходимости разрабатываются меры по увеличению продаж

## Реализованные артефакты

### BPMN-диаграммы
- **AS-IS процесс** - текущее состояние с ручными операциями
- **TO-BE процесс** - автоматизированный вариант с использованием:
  - **Компенсирующих транзакций** для отката ошибочных операций
  - **Параллельных потоков** для одновременной проверки данных
  
### UML-диаграммы
- **Use Case Diagrams** - цели, функции и требования системы
- **Activity Diagrams** - декомпозиция макропроцессов
- **Sequence Diagrams** - взаимодействие между системами
- **State Diagrams** - жизненный цикл объектов
- **User Interface Diagram** - прототипы интерфейсов для всех функций

## Технические особенности реализации

### BPMN-элементы:
- **Компенсирующие обработчики** для отмены уже выполненных финансовых операций при обнаружении ошибок
- **Таймеры событий** для автоматического контроля сроков подготовки отчетности
- **События-ошибки** для устранения неудовлетворительных результатов
- **Условные потоки** для маршрутизации различных типов отчетности

### Используемые инструменты
- **Enterprise Architect** - для комплексного моделирования UML
- **Draw.io** - для быстрого прототипирования BPMN-диаграмм
